# 11. MVP-0.5 Test Doc (黑盒视角)

> **本文档作用**: 给实现者 / reviewer / CI 用。所有 test case 从 `11-mvp-0.5-public-surface.md` 推断出来，**不读 spec / 不读 source**。
> **测试框架**: vitest（DEV-DOC 已锁定）。
> **覆盖目标**: behavior coverage 完整——每个 Behavior + Failure mode + 错误码至少有一个 test 命中。line coverage 不作硬要求。

---

## §0 写作原则

1. **黑盒 only**: 本 doc 仅基于 `11-mvp-0.5-public-surface.md` 列出的函数签名、行为契约、失败模式与错误码推断 test。所有 test case 只验 **external behavior**，不假设 internal implementation（不假设具体 HTTP 框架、不假设具体 yaml lib、不假设具体 LLM 包装方式）。
2. **External-only assertions**: test 断言对象限定为 public interface 暴露的字段（`LLMSubprocResult`, `SanityReport`, `WatchdogState`, `RepoRef`, `PRComment`, `StatusCheck` 等）+ 抛错码 + 文件/网络副作用（mock 出来观测）。
3. **Mock 策略**: 涉及外部 I/O（OS keychain / GitHub API / 文件系统 / LLM 子进程）一律用 test double。本 doc 给出 mock 行为契约，不指定 mock 库实现。建议工具：`memfs` (FS) / `nock` 或 fetch-mock (HTTP) / 一个 keychain abstraction stub / 一个 `LLMSubproc` test double。
4. **命名**: `T-<section>-<behavior>-<variant>`。每个 test ID 必须能在 §"Coverage Map" 反查到 Behavior ID。
5. **不写真代码**: 每个 test case 给 `describe / it / arrange / act / expect` 伪代码骨架，让实现者写 vitest 真实测时不必猜断言点。
6. **范围**: 本 doc 不覆盖性能 / load / 安全渗透 / accessibility——这些走单独 doc。
7. **诚实**: §H 列出当前黑盒视角无法 100% 验证的契约 + 推荐 fallback。

---

## §G Test runner / coverage expectations

### G.1 Runner

- **Framework**: vitest（DEV-DOC 锁定）。
- **命令**: `npm test`（仓库 root），细分入口建议:
  - `npm test -- src/__tests__/cross-engine` (§A)
  - `npm test -- src/__tests__/sanity-gate` (§B)
  - `npm test -- src/__tests__/github` (§C)
  - `npm test -- src/__tests__/watchdog` (§D)
  - `npm test -- src/__tests__/integration` (§E)
- **PBT**: `vitest --reporter=verbose src/__tests__/property` (§F)

### G.2 Mock libraries (recommended, not mandated)

| 用途 | 建议工具 | 备注 |
|---|---|---|
| Filesystem | `memfs` | 不真碰 docs/dogfood/, .git/, ~/.d2p/ |
| HTTP | `nock` / `msw` / `undici.MockAgent` | 拦截 api.github.com；任何 non-allowlisted host 一律拒绝 |
| Keychain | local stub object (`{ get, set, delete }`) + DI | 不调真 keytar / Credential Manager |
| LLM 子进程 | 自写 `LLMSubproc` test double (impl public interface) | 不真 spawn claude/codex/gemini |
| Time | `vi.useFakeTimers()` | 测 backoff、stall threshold、retry 间隔 |
| Property | `fast-check` | 仅 §F |

### G.3 Coverage 目标

- **Behavior coverage**: 每个 `B-X-Y` 必须有 ≥1 个 test ID 命中（见 §Coverage Map）。
- **Failure mode coverage**: §A.5 / §B.5 / §C.4 / §D.7 每行 ≥1 个 negative test。
- **错误码 coverage**: 8 个 `E_*` 每个 ≥1 个触发场景。
- **Line coverage**: 不强制具体百分比；behavior 全绿即视为合格。
- **Flake budget**: 0。任一 test 用真网络 / 真 keychain / 真子进程都视为 critical bug。

### G.4 CI gate

- PR 必须跑 §A-§E 全绿。
- §F (PBT) 跑 200 iterations，failure 必须先 reduce 到 minimum case 再 fix（不允许 `.skip`）。
- §E.4 / §E.5 在每个 PR 必跑（privacy 红线 regression detect）。

## §H 还没法纯黑盒验的契约（诚实承认）

下表列 public surface 里**单元/集成测覆盖不到 / 覆盖到但只能验逻辑路径**的契约。建议 fallback 见每行。

| 契约 | 为什么难黑盒验 | 推荐 fallback |
|---|---|---|
| `estimated_cost_usd` 与 LLM provider 实际计费数额误差 (B-A-7 实质 "准确" 部分) | 真实费率随时间漂移；不调真 API 没法对账 | (1) 单测验「非负 + 单调随 token 数增长」；(2) manual quarterly check 抽 10 个真 session 对账 Anthropic / OpenAI 后台账单；(3) cost-drift alert (差异 > 20% 报警) |
| daemon 启动时确认 2 家 CLI 可用 (B-E-1) | 真子进程行为依赖宿主机 PATH + CLI 版本；CI 不一定装 codex + claude 实体 | 单测验逻辑路径（mock isAvailable）+ daemon smoke 用 docker fixture 装 fake CLI shim |
| "cross-engine 真能抓 collusion" (B-A-6 隐含 spirit) | 是统计性质不是函数契约；单 prompt 测出来的 metadata diff 不等于真能挡共谋 bug | calibration check (B-A-8/9) 已是 fallback；推荐每月跑一次 100-fixture calibration 报 drift |
| 直接调 SDK 红线 (B-E-2) | 静态扫 bundle 只能查显式 import；混淆 / dynamic require 漏检 | (1) bundle string 扫 + import map 检查；(2) network mock 在所有测试 baseline 拒绝 non-allowlisted host (T-C-12 / T-E-5 间接守) |
| `Network 可达` 在 isEnabled (B-C-2) | 网络抖动不稳定；mock 出来的 "可达" 不等于生产可达 | mock 验逻辑路径 + canary post-deploy 检查 `api.github.com` HEAD |
| Keychain 真行为 (B-C-9/10) | 各 OS 实现不同；CI 多为 headless Linux 无 libsecret，难真测 | mock 验调用契约；platform-specific smoke 在 Mac / Win11 / Linux+libsecret 各一台 host 跑 acceptance |
| GitHub 真 rate limit response shape (T-C-fail-rate-limit) | GitHub 偶尔改 header；mock 出来的可能与真不符 | mock 验逻辑 + 月度 read-only canary 命中 rate limit 后看 response shape 是否漂 |
| 子进程被 kill 后无 orphan (T-A-4) | Node child_process kill 信号在 Win vs POSIX 行为不同；mock 测不到真 orphan | mock 验调用 kill；platform smoke 用真 sleep 子进程验 PID 在 timeout 后被回收 |

| Dogfood-debt mode 维护者 postmortem 流程 (D.5 exit 条件) | 是流程契约不是代码契约 | postmortem 模板 lint + PR template check；不写测 |
| Cost cap 真触发后 graceful degradation 报告内容质量 (B-E-6) | 单测验 shape，不验 "用户能看懂" 质量 | UX review + 5 个真用户 dogfood 反馈 |

1. Quarterly manual cost reconciliation (1)
2. Monthly cross-engine calibration run (3)
3. Platform-specific smoke matrix (Mac / Win / Linux+libsecret) for keychain + subproc (6, 8)
4. GitHub canary read-only probe (5, 7)
5. Repo settings audit checklist (9)
