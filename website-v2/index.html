机制 · 05 为什么是 4 道 reviewer · 不是 1 个

每个 fix 走 4 道门 才有机会合 main。

单一 LLM critic 是常见做法,但 LLM 自审的盲点也是公认的——它判不出自己的盲区。ZeroU 用 4 个角色 + 不同 fresh context 把这道盲区切碎。任一道不过,整 fix 回退或升级 NEED_HUMAN。

  1. G1.

    不是 AI · 是命令行。

    tsc --noEmit · npm test · npm run build 。先用你的项目自己的 check 命令跑一遍 — 跑不过的 fix 根本进不了 reviewer。

    static gate
  2. G2.

    对题快检 · 不偏题、不 scope creep。

    haiku 模型 quick pass:这个 fix 写的是 vision 想要的吗?有没有顺手改了别的?score < 0.7 自动驳回。

    alignment
  3. G3.

    思路审核 · approach 扎不扎实。

    sonnet,fresh context,从零看 diff。verdict 四档:APPROVE / RETRY_WITH_HINTS / ROLLBACK / SPLIT_INTO。失败给重试提示。

    behavioral
  4. G4.

    攻击视角 · 它能被打穿吗。

    高敏 category (auth / sql / crypto) 才触发。sonnet 出 N 条 attack vector,一条 BREAK 整 fix 就 rollback。

    adversarial
不可撤销 · 06 v1 不会让步的五条

5 件不可撤销的事。

改任意一条意味着 ZeroU 不再是 ZeroU。

  1. 01.

    双绿才停。

    preset 全部 done + done-check agent 判定 vision verdict YES。少一个绿都不 SESSION_DONE。

  2. 02.

    NEED_HUMAN 写进 PR body。

    reviewer 拒掉的 gap 不藏。每个 PR 自带"它做不到的事"清单 + reason code。你 review 的是诚实的 delta。

  3. 03.

    4 道 reviewer 一道不能少。

    不是配置选项。static / alignment / behavioral / adversarial 全开 — 这是它存在的理由。

  4. 04.

    本地 daemon · 代码不离机。

    不是云服务。daemon 在你电脑上跑,worktree 在你磁盘上写,merge 在你本地 git。我们手里没有你的代码。

  5. 05.

    全开源 · 你的 LLM key。

    MIT · 自带 Anthropic / OpenAI-compat key · 零后端 · 没有 SaaS 转身位。删了就是全删,我们手里不留东西。
